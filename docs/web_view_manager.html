WebViewManager Detailed Design

The WebViewManager class is responsible for creating and configuring the WKWebView, adding the webView to the parent view, setting up constraints, and injecting scripts.

1. Initialization

WebViewManager will be initialized with a reference to the parent UIViewController and the WKWebViewConfiguration.

Properties:

  • parentViewController: UIViewController
  • webViewConfiguration: WKWebViewConfiguration

2. Create and configure WKWebView

WebViewManager will create a WKWebView instance using the provided configuration, set up the WKWebView's properties, and add the webView as a subview to the parent view controller's view.

Methods:

  • createWebView(): WKWebView

3. Add webView to parent view and set up constraints

WebViewManager will add the created webView as a subview to the parent view controller's view and set up constraints.

Methods:

  • addWebViewToParentView(webView: WKWebView)

4. Inject JavaScript scripts

WebViewManager will inject the required JavaScript scripts into the webView using the webView's userContentController. This includes adding user scripts and script message handlers.

Methods:

  • injectJavaScriptScripts(webView: WKWebView)

By implementing the WebViewManager, we can separate the concerns related to the webView's creation, configuration, and setup from the ViewController, adhering more closely to the SOLID principles.
