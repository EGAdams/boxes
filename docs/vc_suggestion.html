Refactoring Plan for ViewController

Here is the plan to break up the remaining parts of the ViewController into simpler objects:

1. WebViewManager

This class will handle the creation and setup of the WKWebView, including adding the view to the parent view, setting up constraints, and injecting scripts.

Responsibilities:

  • Create and configure WKWebView
  • Add webView to parent view and set up constraints
  • Inject JavaScript scripts

2. FileLoadingManager

This class will be responsible for handling the files to download and the loading of the files into the webView.

Responsibilities:

  • Manage the files to download
  • Load files into webView after download

3. JavaScriptMessageHandler

This class will handle JavaScript messages received from the webView.

Responsibilities:

  • Implement JavaScriptInterfaceDelegate protocol
  • Handle messages received from JavaScript

By breaking up the ViewController into these separate objects, we can achieve better separation of concerns and adhere more closely to the SOLID principles.
